Synopsis

The objective for this assignment is to develop predictive models to predict the manner in which the subjects did the exercises. The quality of the exercise is represented by the classe variable in the training set. The data provided for this project consists of the sensor readings from 6 participants who performed 10 repetitions of Unilateral Dumbbell Biceps Curl in five different fashions: exactly according to the specification (Class A), throwing the elbows to the front (Class B), lifting the dumbbell only halfway (Class C), lowering the dumbbell only halfway (Class D) and throwing the hips to the front (Class E). Class A corresponds to the specified or correct execution of the exercise, while the other 4 classes correspond to common mistakes.

During this project, we explore, clean-up and understand the data to determine what specific predictor variables are likely to influence on the prediction on “classe” variable which represent exercise quality and takes on values of “A, B, C, D, or E”. We remove all the columns from the data set which are non-sensory data and only identify the different participants and timestamps of the data collections. Additionally, we also remove all the data columns which are empty or have NA values. We then conduct exploration on the remaining data to identify any predictor variables which does not show any relationship with the response variable “classe”. We specifically could not identify any such variable in the limited tries and thus decided to leave all the variables, 52 of them to fit the model.

We first fit the random model using the caret package which provides a wrapper when making call to randomForest along with many functions and options to preprocess and fine tune the model. We next fit the random model by making direct call to randomForest function from randomForest package. We compare the results from the two different models as well as the time it takes to fit two models.

We find that direct call to randomForest gives us a better Out-of-sample Error Rate and a time to fit the model which is faster than the caret package approach. This is as expected since caret package uses a wrapper to make a call to randomForest function with its own overhead, etc. On the other hand, we expect model fitted with caret package to perform with a larger variety of data situations since we expect to have a better overall model.

Create training and validation data sets

For applying the classification models, we explicitly create a training set for fitting the model and hold back part of the data as a validation set for model validation. Fitted models will be trained on the training data set and improved and tuned using results of cross-validation. Then we take the final fitted model run it against the validation data to calculate out-of-sample error rate or misclassification rate. The model has not seen the validation data before and running against this data we get unbiased estimate of the error rate of the fitted model.

We take the provided training data set (which has been cleaned and preprocessed as described earlier) and divide the data into two sets - 60% into training set and 40% into validation set.

Model fitting using caret package and randomForest

First we use the caret package and a call to random forest method to fit the model. We use the trainControl function to set the tuning parameter. Default method for resampling is bootstrap resampling. We try several approaches to cross-validation using options in trainControl and settle on K-fold cross validation with K = 3. Using the allowParallel option, we also take advantage of multiple cores in the computer to run the model. We achieve a reasonable run time of about 7-8 minutes through this approach compared to multiple hours with the initial approaches.

set.seed(33833)
ptm <- proc.time()

fitcontrol <- trainControl(method = "cv", number = 3, allowParallel = TRUE)

model.caret.rf <- train(as.factor(classe) ~ ., data = trainData , method="rf",
    trControl = fitcontrol, importance = TRUE, proximity = FALSE)

Summary

We fit the model to the provided training data set using random forest technique - first by making the call through the caret package and using the tuning and preprocessing options available through that package and then by making the direct call to the randomForest function from the randomForest package. We find that the performance is much better with the direct call to randomForest function but we expect the model created through caret package to perform better with any other data values which were not part of the training set.

We used both models later to predict the classe variables from the testing data set, pml-testing.csv and achieved 20/20 correct predictions in both cases for the second part of the course assignment.
